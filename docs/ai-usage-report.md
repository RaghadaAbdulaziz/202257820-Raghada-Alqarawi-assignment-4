# AI Usage Report

## Tools Used & Use Cases

### Claude
Claude was used as the primary tool for code generation and debugging. It assisted in:
- Writing and improving JavaScript functions for dynamic features such as project filtering, search functionality, and modal interaction.
- Reviewing and optimizing existing code structure to ensure better organization and readability.
- Debugging issues related to event handling and DOM manipulation.

### Google Gemini
Google Gemini was used mainly for documentation support and UI/UX enhancement. It assisted in:
- Structuring the README and technical documentation.
- Suggesting improvements for user experience, such as clearer interaction guidance and better form structure.
- Providing recommendations for layout consistency and visual clarity.

### ChatGPT
ChatGPT was used as a supporting tool to refine implementation details and improve overall project quality. It assisted in:
- Integrating the external API (Developer Advice feature)
- Improving state management using localStorage
- Enhancing form validation and user feedback
- Refining instructions and fixing deployment issues on GitHub Pages

---

## What AI Generated
AI helped generate parts of the HTML structure, CSS styling ideas, and JavaScript logic for features such as filtering, modal interaction, API integration, and form validation.

## What Was Rejected
Some AI suggestions were not used because they introduced unnecessary complexity, external libraries, or features beyond the assignment requirements.

## What Was Modified
Most AI-generated outputs were modified to:
- Match the assignment requirements
- Simplify the implementation
- Ensure compatibility with GitHub Pages deployment

## Reason for Changes
Changes were made to keep the project simple, functional, and aligned with the required technologies (HTML, CSS, JavaScript only).

## AI Hallucination
In some cases, AI suggested features or implementations that were not needed or assumed requirements that were not part of the assignment. These were identified and removed.

## What AI Misunderstood
AI occasionally suggested advanced solutions or frameworks, while the assignment required a simpler implementation using only basic web technologies.

## Domain Knowledge AI Lacked
AI did not fully understand the specific constraints of the assignment, such as limiting the use of external tools and focusing on core front-end concepts.

---

## Benefits & Challenges

### Benefits
Using AI significantly improved development efficiency. It helped transform the portfolio from a static website into a more interactive application by suggesting dynamic features such as filtering, searching, and modal popups. It also improved code readability and reduced debugging time.

AI tools also enhanced the user experience by recommending clearer interaction flows and better UI structure.

### Challenges
One challenge was ensuring that AI-generated solutions remained simple and aligned with assignment requirements. Some suggestions included unnecessary complexity or external libraries.

This was resolved by refining prompts and focusing only on standard HTML, CSS, and JavaScript solutions.

---

## Learning Outcomes

Through using AI tools, I learned how to:
- Implement dynamic UI features using JavaScript (filtering, searching, sorting, modal systems)
- Integrate external APIs using `fetch()` and handle errors properly
- Manage application state using `localStorage`
- Improve user experience by adding clear instructions and real-time feedback

---

## Responsible Use & Modifications

Although AI tools assisted in generating and improving code, all outputs were reviewed, tested, and modified before use. The final implementation reflects my own understanding and adjustments.

All features were implemented using standard web technologies without external frameworks, ensuring the project meets the assignment requirements.
